Dùng kiến trúc này là gọn nhất cho project hiện tại:

Internet
  -> ALB
    -> ECS Fargate Task
       - frontend nginx container :80

Postgres không push lên ECR. ECR chỉ chứa frontend và backend. AWS ECR dùng để push/pull Docker image, còn database nên dùng RDS PostgreSQL để có storage, backup, restore ổn định hơn. AWS docs: ECR push image, RDS create DB, ECS env variables.

1. Chuẩn bị local
Ở D:\website-exam:

aws configure
aws sts get-caller-identity
docker version
Chọn region, ví dụ:

$env:AWS_REGION="ap-southeast-1"
$env:AWS_ACCOUNT_ID="<account_id_cua_m>"
2. Tạo 2 repo ECR

aws ecr create-repository --repository-name v-exam-backend --region $env:AWS_REGION
aws ecr create-repository --repository-name v-exam-frontend --region $env:AWS_REGION
Login ECR:

aws ecr get-login-password --region $env:AWS_REGION | docker login --username AWS --password-stdin "$env:AWS_ACCOUNT_ID.dkr.ecr.$env:AWS_REGION.amazonaws.com"
3. Build và push image

Backend:

docker build -t v-exam-backend .
docker tag v-exam-backend:latest "$env:AWS_ACCOUNT_ID.dkr.ecr.$env:AWS_REGION.amazonaws.com/v-exam-backend:latest"
docker push "$env:AWS_ACCOUNT_ID.dkr.ecr.$env:AWS_REGION.amazonaws.com/v-exam-backend:latest"
Frontend:

docker build -t v-exam-frontend .\frontend
docker tag v-exam-frontend:latest "$env:AWS_ACCOUNT_ID.dkr.ecr.$env:AWS_REGION.amazonaws.com/v-exam-frontend:latest"
docker push "$env:AWS_ACCOUNT_ID.dkr.ecr.$env:AWS_REGION.amazonaws.com/v-exam-frontend:latest"
4. Tạo RDS PostgreSQL
Trong AWS Console:

RDS -> Create database
Engine: PostgreSQL
Template: Dev/Test trước đã
DB name: v_exam_hub
Username: ví dụ exam_admin
Password: tự đặt mạnh
Public access: nếu làm nhanh để test có thể bật tạm, nhưng production nên để private
Security Group RDS: chỉ cho ECS security group truy cập port 5432
Sau khi tạo xong, lấy endpoint dạng:

xxx.ap-southeast-1.rds.amazonaws.com
DB URL production sẽ là:

host=<rds-endpoint> port=5432 user=exam_admin password=<password> dbname=v_exam_hub sslmode=require
5. Import schema/data vào RDS
Nếu RDS public tạm thời và máy m truy cập được:

Nếu RDS private, cách sạch hơn là dùng CloudShell/EC2 bastion trong cùng VPC rồi chạy psql từ đó.
